# AIAA3201 Final Project - Video Super-Resolution

This repository contains the full AIAA3201 video super-resolution project, organized into three parts:

- Part1: classical baselines and a lightweight SRCNN pipeline
- Part2: BasicVSR-based video super-resolution with two-stage GAN fine-tuning
- Part3: research exploration and reliability-guided fusion experiments

The repo is kept in a clean structure so each part can be run independently while still sharing a common dependency base.

## Repository Layout

```text
Part1/                  # SRCNN, spatial baselines, temporal baseline, report pipeline
Part2/                  # BasicVSR, GAN training, inference, evaluation, weights
part3/                  # Part 3 exploration code and selected results
datasets/               # Dataset storage
results/                # Top-level result artifacts and visuals
requirements.txt        # Unified dependency list for all parts
```

## Environment Setup

Install the shared dependencies from the project root:

```bash
pip install -r requirements.txt
```

The unified dependency list covers Part1, Part2, and Part3. Part3 also relies on diffusion-related packages such as `diffusers`, `transformers`, `accelerate`, and `safetensors`.

## Part1 Overview

Part1 provides classical and lightweight baselines for video super-resolution:

- spatial comparison: Bicubic, Lanczos, and SRCNN
- temporal baseline: weighted multi-frame averaging with optional sharpening
- report pipeline: one-command generation of CSV summaries, preview images, and a markdown report

Main entry points:

```bash
cd Part1
python train_srcnn.py
python eval.py
python temporal_baseline.py
python infer_srcnn.py
bash run_part1_report.sh
```

Important outputs:

- `checkpoints/srcnn_best.pt`
- `results_part1.csv`
- `temporal_metrics.csv`
- preview images and report summaries generated by the report workflow

## Part2 Overview

Part2 implements a BasicVSR-based VSR pipeline with a two-stage training strategy:

1. stage 1: pixel-level reconstruction with L1 / Charbonnier loss
2. stage 2: GAN fine-tuning with perceptual and adversarial losses

Main components:

- `model_basicvsr.py` for the core bidirectional propagation network
- `model_spynet.py` for optical flow estimation
- `model_discriminator.py` and `loss_gan.py` for GAN training
- `vsr_dataset.py` for Vimeo90K sequence loading
- `train_vsr_stage1.py`, `train_vsr_gan.py`, `train_vsr_unified.py`
- `inference_vsr.py`, `eval_vsr.py`, `visual_compare.py`, `frames_to_video.py`

Typical workflow:

```bash
cd Part2
python train_vsr_stage1.py
python train_vsr_gan.py
python eval_vsr.py
python inference_vsr.py
```

Expected outputs include model checkpoints in `weights/`, evaluation results in `evalresult/`, and visual comparisons in `visual/`.

## Part3 Overview

Part3 is the exploration and extension track. It is organized by research direction under `part3/scripts/`:

- `direction_a_temporal_refinement/`
- `direction_b_generative_prior/`
- `direction_c_main_fugr/`
- `exploratory_legacy/`

Main themes:

- Direction A: temporal refinement and train-time temporal fine-tuning
- Direction B: generative prior and frequency-constrained fusion analysis
- Direction C: reliability-guided fusion and the main FUGR-style reconstruction path

Typical usage:

```bash
cd part3/scripts
python direction_a_temporal_refinement/directionA11_train_temporal_refiner.py
python direction_b_generative_prior/directionB_frequency_fusion.py
python direction_c_main_fugr/part3_fugr_vsr.py
python direction_c_main_fugr/eval_fugr_fullval.py
```

Part3 keeps selected results under `part3/results/`, figures under `part3/figures/`, and run logs under `part3/logs_selected/`.

## Data and Large Files

The project assumes Vimeo90K-style training and validation data. Large assets such as pretrained weights, generated videos, and raw experiment folders are intentionally not fully tracked in the repository.

If you are running the project locally, make sure your dataset paths and output directories match the arguments used by each script.

## Suggested Reading Order

If you want the fastest path through the project, read it in this order:

1. Part1 for the classical baseline pipeline
2. Part2 for the stronger BasicVSR + GAN system
3. Part3 for the research extensions and the final best-performing reconstruction path
